Yet another (S)CSS / jQuery component library! Since 2014 Codebase has occasionally been added to and improved. An earlier version had another name â€“ until I hit upon the idea of calling my code base “Codebase”.

As of version 1.0 Codebase follows the BEM naming methodology for its mixins and classes.

All layout, utilities and components have been created first as @mixins and then classes have been built from these mixins.

So although the docs demonstrate the Codebase classes, SCSS users can also @include Codebase mixins in their own mixins and classes.

All the Codebase !default variables can be easily overridden.
