Basics

Starting from an @import of normalize.css and building upon it, Codebase adds some basic styling and other interestng features.

Some of these Basics examples have been built first as Pug mixins, in order to be useful in rapid prototyping or style guide composition.

(Some basics really can't be more rapidly created by calling a mixin than by actually hand-coding them. E.g. there are no mixins for horizontal rules, floats, clearfix, badges, labels, text columns or enlarged typography.)

Headings

By default, h1 has zero top margin whereas h2-h6 headings ordinarily have top margin equal to 2× the default line height, i.e. 48px, or 6x element grid units. (This top margin is removed in header pairings.)

Headings h1-h4 are given increasted font-size for medium screens up. All headings are rendered font-weight: normal so that you can choose to emphasize some word(s) in bold.

Enlarged Typography

By defult, Codebase has its font sizes for everything based on a set of $base-font-* variables. But there is also a set of $enlarged-font-* variables. (You can override both the base and enlarged variables because each are set as !default variables.)

There are three ways to use the enlarged typography in Codebase:

1. Toggle enlarged typography

You can toggle on the enlarged typography everywhere, by overriding one variable from false to true: 

$enlarged-typography-everywhere: true !default;

2. Overriding enlarged typography defaults

You can override the $enlarged-font-size and $enlarged-line-height default variables within your theme (see Getting Started: Advanced Setup).

Overriding these variables will affect everything (if you toggle on enlarged typography) or only stuff contained in a .enlarged (see below).

3. Enlarged typography wrapper class

Codebase can render enlarged typography simply by placing any Codebase-styled HTML in a wrapper that has class .enlarge.

This may be useful e.g. for long-read articles.

Tables

Simply add class .tableto tables. This adds the styles that you see below.

The codebase.js adds two wrappers around tables that enables responsive overflow by horizontal scrolling: table-wrap--outer and table-wrap--inner. (Note: you don't need to add these wrappers yourself.)

Also, if the table width is wider than the width allowed in the webpage layout, then an inset box shadow is added to indicate that there is more to the table than what is visible.

Form Elements

Form elements expand to 100% of their container width. Give them structure by wrapping them in grid columns.

There is no added margin or padding around form elements – so nothing you need to override. Add whatever margin or padding (or not) as suits your own design.

Images

By default, images are inline elements, sitting on the baseline of any text they are share a block element (e.g. paragraph) with. Codebase also gives them code max-width: 100% so that large images are "repsonsive", shrinking to fit in containers (or windows) narrower than the actual image width.

Further styling can be added using decoration classes (e.g. borders).
